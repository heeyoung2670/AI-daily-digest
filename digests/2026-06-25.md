## 🧪 연구 및 논문 동향

### A Technical Taxonomy of LLM Agent Communication Protocols
분산 에이전트 네트워크에서 LLM 에이전트 간 통신 프로토콜을 체계적으로 분류한 연구. 에이전트 수가 늘어날수록 통신 규약의 표준화 없이는 조율 실패가 발생한다는 점을 실증적으로 보여주며, MCP와 같은 프로토콜 레이어의 필요성을 뒷받침한다.
- 출처: https://arxiv.org/abs/2606.19135

### GRPO Does Not Close the Multi-Agent Coordination Gap
강화학습 방법론 중 하나인 GRPO(Group Relative Policy Optimization)를 멀티에이전트 조정 문제에 적용했을 때 기대만큼 성능이 향상되지 않는다는 결과를 제시한 논문. 에이전틱 AI의 협력·조율 능력 향상에 강화학습만으로는 한계가 있음을 보여준다.
- 출처: https://arxiv.org/abs/2606.07845

## 🛠️ 개발자 도구 및 오픈소스

### Apple Foundation Models — fm CLI + Python SDK로 일반 개발자까지 개방
WWDC 2026에서 공개된 `fm chat` CLI는 Swift나 Python 코드 없이 터미널에서 바로 Apple 온디바이스 모델을 호출할 수 있게 해준다. python-apple-fm-sdk는 Python 3.10 이상 환경에서 Apple Silicon Mac을 서버처럼 활용할 수 있는 길을 열었다. 여름 중 Linux 서버 지원이 추가되면 Apple 생태계 밖 개발자도 Apple AI를 백엔드로 쓸 수 있게 된다.
- 출처: https://developer.apple.com/videos/play/wwdc2026/241/

### SkillAxe — 에이전트 스킬 자동 개선 프레임워크 공개
LLM이 스스로 작성한 에이전트 스킬을 평가 기반으로 반복 정제하는 SkillAxe 프레임워크가 arXiv에 공개됐다(2606.10546). 사람의 피드백 없이 에이전트가 자율적으로 도구 사용 능력을 향상시키는 방향을 제시해, 에이전틱 AI 자기개선 연구의 새 방향으로 주목받고 있다.
- 출처: https://arxiv.org/abs/2606.10546

### OpenCode — 오픈소스 코딩 에이전트, GitHub 스타 16만 돌파 지속
모델 무관 오픈소스 코딩 에이전트 OpenCode가 160K+ 스타, 월 활성 개발자 750만 명을 기록 중이다. 75개 이상의 모델 프로바이더를 지원하며, LSP 통합으로 컴파일러 오류를 모델에 실시간으로 전달한다. SpaceX의 Cursor 인수 이후 오픈소스 대안으로서의 존재감이 더욱 부각되고 있다.
- 출처: https://www.devflokers.com/blog/open-source-ai-roundup-june-2026
