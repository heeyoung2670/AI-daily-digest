## 🧪 연구 및 논문 동향

### Predictive Maps of Multi-Agent Reasoning (arXiv:2605.11453)
멀티에이전트 LLM 시스템 배포 전 최적 통신 토폴로지(chain·star·mesh 등)를 진단하는 프레임워크를 제안했다. 후계 표현(Successor Representation)을 기반으로 폐쇄형 스펙트럼을 도출해, 어떤 구조가 특정 작업에 유리한지 추론 전에 예측할 수 있다. 멀티에이전트 시스템 아키텍처 결정에 실용적인 기준을 제공한다.
> 출처: https://arxiv.org/abs/2605.11453

### Safe Multi-Agent Behavior: Constraint Drift 문제 (arXiv:2605.10481)
멀티에이전트 LLM 시스템에서 안전 제약이 시간이 지남에 따라 점진적으로 완화되는 'Constraint Drift' 현상을 분석했다. 안전 속성은 초기 주입만으로는 부족하고 지속적으로 유지돼야 한다는 Constraint State Governance 패러다임을 제안한다. 에이전트를 프로덕션에 배포하는 팀이라면 주목할 연구다.
> 출처: https://arxiv.org/abs/2605.10481

### MARLIN: 게임이론 RL로 LLM 인퍼런스 지속가능성 최적화 (arXiv:2605.13496)
클라우드 데이터센터에서 LLM 인퍼런스 시 응답 속도(TTFT)·탄소 배출·수자원 사용·에너지 비용을 동시에 최적화하는 멀티에이전트 강화학습 프레임워크다. 비용과 환경 영향을 함께 줄이는 그린 AI 연구의 최전선에 있으며, 대규모 AI 인프라 운용 팀에 시사점이 크다.
> 출처: https://arxiv.org/abs/2605.13496

### Goose + Qwen3-Coder로 완전 무료 로컬 코딩 에이전트 구성
Block의 오픈소스 에이전트 Goose와 Ollama·Qwen3-Coder를 조합해 클라우드 없이 로컬에서 동작하는 코딩 에이전트를 구성할 수 있다. 구독료·사용량 한도 없이 완전 무료이며, Llama 4 Scout 등 다른 로컬 모델과도 연동 가능하다. GitHub에서 27K 스타를 기록하며 Claude Code 대안으로 주목받고 있다.
> 출처: https://github.com/aaif-goose/goose

### Claude Code, SWE-bench 80.9% 달성 + Agent Teams 기능 추가
Claude Code가 SWE-bench에서 80.9%를 기록하며 코딩 에이전트 새 기준을 세웠다. 5월 말 추가된 Agent Teams 기능으로 병렬 서브에이전트가 코드베이스 여러 부분을 동시에 작업할 수 있다. Effort Control 슬라이더로 응답 속도와 추론 깊이를 조절해 단순 작업과 복잡한 작업 모두 효율적으로 처리한다.
> 출처: https://newsletter.pragmaticengineer.com/p/ai-tooling-2026

### Google Turbo Quant — KV 캐시 메모리 오버헤드 획기적 절감 알고리즘
Google 연구팀이 LLM 추론 시 KV 캐시 메모리 부담을 크게 줄이는 Turbo Quant 알고리즘을 공개했다. VRAM이 제한된 환경에서 장문 컨텍스트 모델을 효율적으로 배포할 수 있게 해, 엣지 디바이스와 소규모 GPU 클러스터에서의 LLM 실행을 현실적으로 만든다.
