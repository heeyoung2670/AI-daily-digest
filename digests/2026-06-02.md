### 01. Microsoft Build 2026 개막 — Windows가 AI 에이전트 플랫폼으로 전환
오늘 샌프란시스코 Fort Mason Center에서 Microsoft Build 2026이 개막했다. Satya Nadella 기조연설의 핵심은 Windows를 AI 에이전트 실행 플랫폼으로 전환하는 것. Copilot Workspace가 정식 GA로 전환되어 자연어로 버그·기능을 지시하면 Copilot이 파일을 수정하고 PR을 자동 생성한다. Foundry Local도 Windows·macOS Apple Silicon·Linux x64 전면 GA를 선언, 클라우드 없이 로컬에서 AI 추론이 가능해졌다.
- 출처: https://windowsnews.ai/article/build-2026-microsofts-platform-shift-to-ai-agents-copilot-and-azure-ai-foundry-takes-center-stage-in.420960

### 02. GitHub Copilot, 8월부터 Microsoft 자체 모델 'Project Polaris'로 전환
Build 2026에서 Microsoft는 자체 개발 AI 코딩 모델 **Project Polaris**를 발표했다. 8월부터 GitHub Copilot 기본 모델을 GPT-4 Turbo에서 Polaris로 교체하며, 3개월 유예 기간을 제공한다. 벤더 종속을 줄이고 비용 구조를 개선하겠다는 포석으로, Copilot을 OpenAI로부터 독립시키는 '프로젝트 오리온'의 연장선이다.
- 출처: https://chatforest.com/builders-log/microsoft-build-2026-recap-windows-agent-platform-project-polaris-copilot-workspace/

### 03. OpenAI, GPT-4.5 ChatGPT 6월 27일 종료 — GPT-5.5가 기본 모델로 안착
OpenAI가 GPT-4.5의 ChatGPT 서비스 종료일을 6월 27일로 확정했다. 4월 23일 출시된 GPT-5.5가 이미 플러스·프로 구독자의 기본 모델로 자리 잡았으며, 코딩·연구·데이터 분석에서 전작 대비 큰 도약을 보여주고 있다. 5월에는 응답 속도를 높인 GPT-5.5 Instant도 출시됐다.
- 출처: https://help.openai.com/en/articles/6825453-chatgpt-release-notes

### 04. Google·Meta, 개인 AI 에이전트 경쟁 가속 — 'Gemini Spark'와 'Hatch'

- 출처: https://the-decoder.com/google-and-meta-race-to-build-personal-ai-agents-as-anthropic-and-openai-pull-further-ahead/

xAI가 Grok 4.4(1조 파라미터)를 2~3주 내에, 이어서 Grok 4.5(1.5조 파라미터)를 4~5주 내에 출시할 예정이라고 밝혔다. 현재 Grok 4는 SuperGrok·Premium+ 구독자와 xAI API를 통해 제공 중이며, 5월에는 무한 캔버스 기반 이미지·영상 생성 에이전트 'Grok Imagine Agent Mode' 베타도 출시됐다. 2M 토큰 컨텍스트 지원으로 장문서 분석에 강점을 가진다.
- 출처: https://x.ai/news/grok-4

## 🧪 연구 및 논문 동향

### USEagent: 통합 소프트웨어 엔지니어링 에이전트 (arXiv:2506.14683)
ICSE '26 발표 논문. LLM을 단순 코딩 도구가 아닌 소프트웨어 유지보수·진화 전체를 담당하는 통합 에이전트로 설계했다. 불완전한 패치 수정, 신규 기능 추가, 타인 코드 인수 등 현실적인 복잡 시나리오를 처리할 수 있는 멀티캐퍼빌리티 오케스트레이션이 핵심이다. 기존 특화형 에이전트(테스트·디버깅 전용 등)의 한계를 넘어선 접근법이다.
- 출처: https://arxiv.org/abs/2506.14683

### Ares: 효율적 LLM 에이전트를 위한 추론 노력 적응 선택 (arXiv:2603.07915)
추론 능력을 강화한 대형 모델이 멀티스텝 의사결정에서 뛰어난 성능을 보이지만, 모든 작업에 동일한 추론 비용을 지불하는 것은 비효율적이다. Ares는 작업 난이도에 따라 추론 노력(chain-of-thought 길이 등)을 동적으로 선택해 비용 대비 성능을 최적화한다. 에이전트 운영 비용 감소에 실용적 함의가 크다.
- 출처: https://arxiv.org/abs/2603.07915

### The Thinking Boundary: 멀티모달 작업의 추론 적합성 정량화 (arXiv:2603.04415)
추론 특화 LLM이 멀티모달 시나리오 전반에 걸쳐 실제로 유효한지를 Dual Tuning 방식으로 정량 측정한다. 텍스트 추론에서 효과적인 chain-of-thought가 이미지·비디오 포함 멀티모달 태스크에서는 오히려 역효과를 낼 수 있음을 규명한다. 멀티모달 에이전트 설계 시 추론 모드 선택 기준을 제시한다.
- 출처: https://arxiv.org/abs/2603.04415

## 🛠️ 개발자 소식

### Microsoft Foundry Local GA — 클라우드 없이 Windows·macOS·Linux에서 AI 추론
Build 2026에서 GA를 선언한 Foundry Local은 Windows, macOS(Apple Silicon), Linux x64 전 플랫폼을 지원한다. OpenAI 호환 API 포맷(Chat Completions, Audio Transcription)을 제공해 클라우드와 온디바이스 추론을 코드 변경 없이 전환할 수 있다. Python·JavaScript·C#·Rust SDK가 함께 제공되며, 퍼-토큰 비용 없이 로컬 AI 기능을 앱에 내장할 수 있다.
- 출처: https://devblogs.microsoft.com/foundry/foundry-local-ga/

### Raindrop Workshop — AI 에이전트 로컬 디버거 오픈소스 공개 (MIT)
AI 에이전트 모니터링 스타트업 Raindrop이 오픈소스 로컬 디버거 **Workshop**을 공개했다. 에이전트가 실행하는 모든 토큰·툴 호출·의사결정을 로컬 SQLite DB에 기록하고, localhost:5899 대시보드에서 실시간으로 확인할 수 있다. MCP를 통해 Claude Code에 트레이스를 노출, 코딩 에이전트가 직접 eval을 작성하고 문제를 수정할 수 있다.
- 출처: https://venturebeat.com/technology/developers-can-now-debug-and-evaluate-ai-agents-locally-with-raindrops-open-source-tool-workshop

### Copilot Workspace GA — 자연어로 PR까지 자동 생성
Build 2026에서 GitHub Copilot Workspace가 정식 출시됐다. 개발자가 버그나 기능 요청을 자연어로 설명하면 Copilot이 구현 계획을 수립, 레포지토리 전반의 파일을 수정하고 PR을 오픈한다. 여러 레포지토리를 동시에 이해하는 '워크스페이스 모드'도 지원한다. Project Polaris(자체 모델) 전환과 맞물려 Copilot 생태계의 자율성이 한 단계 높아진다.
- 출처: https://chatforest.com/builders-log/microsoft-build-2026-recap-windows-agent-platform-project-polaris-copilot-workspace/
